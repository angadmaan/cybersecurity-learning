# The Phishing Lifecycle

A typical phishing campaign follows several stages:

1. Target Selection
2. Creating a Fake Identity
3. Building a Fake Login Page
4. Delivering the Phishing Link
5. Victim Interaction
6. Credential Collection
7. Detection and Response

Understanding each stage helps defenders interrupt the attack before sensitive information is compromised.

---

# Why Users Fall for Phishing

Successful phishing attacks often exploit human psychology rather than technical weaknesses.

Common techniques include:

- Creating urgency
- Offering rewards
- Fear-based messaging
- Trusting familiar brands
- Curiosity
- Lack of security awareness

Regular security awareness training reduces the effectiveness of these techniques.

---

# Detecting Phishing Attacks

Security teams identify phishing attempts by looking for:

- Suspicious domains
- Newly registered websites
- Fake login pages
- URL mismatches
- Unusual email headers
- Spoofed sender addresses
- Unexpected requests for credentials

Organizations also use email filtering and domain reputation services to detect malicious campaigns.

# Preventing Phishing

Effective defenses include:

- Multi-Factor Authentication (MFA)
- Email filtering
- Security awareness training
- Strong password practices
- URL verification
- Browser security features
- Endpoint protection
- DNS filtering

No single control eliminates phishing, so organizations implement multiple layers of defense.

---

# Role of Security Analysts

Security Analysts play an important role in phishing defense by:

- Monitoring phishing alerts
- Investigating suspicious emails
- Blocking malicious domains
- Conducting awareness training
- Responding to incidents
- Improving organizational defenses

Understanding attacker techniques helps analysts respond more effectively.

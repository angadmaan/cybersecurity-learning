# Day 13

## Overview

Day 13 was focused on exploring another cybersecurity tool called **Devil's Call**, learning how to install and run tools directly from GitHub repositories, and continuing our work with **UFONet**.

A major part of today's session was troubleshooting errors, understanding tool documentation, and learning how cybersecurity professionals analyze attack techniques in controlled environments to better understand how attackers think and how defenders can protect systems.

---

## Installing Devil's Call

The first activity of the day was installing and running **Devil's Call** from its official GitHub repository.

### Commands Used

```bash
git clone https://github.com/404-ghost/Devil-s-Call
```
```
cd Devil-s-Call
```
```
python3 devilscall.py
```

### What I Learned

* How to clone a project directly from GitHub
* Navigating project directories using the terminal
* Running Python-based security tools
* Understanding the structure of open-source cybersecurity projects

This exercise helped me become more comfortable working with GitHub repositories and Linux command-line environments.

## Understanding DDoS Attacks from a Defensive Perspective

The final discussion of the day focused on understanding the mindset behind **Distributed Denial of Service (DDoS)** attacks.

The goal was not to attack real systems, but to understand:

* How attackers attempt to overwhelm services
* Why websites become unavailable during large-scale attacks
* How security teams identify abnormal traffic
* What organizations do to defend against DDoS attacks

Learning how attacks are performed helps security professionals better understand how to detect, prevent, and mitigate them.

Today's session reinforced an important lesson:

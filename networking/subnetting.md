# Subnetting Basics

## Introduction

Subnetting is the process of dividing a large network into smaller networks called subnets.

It improves efficiency, security, and network management.

---

## Why Subnetting is Needed

Benefits:

- Better performance
- Reduced broadcast traffic
- Easier administration
- Improved security

---

## Network ID and Host ID

An IP address contains:

### Network ID

Identifies the network.

### Host ID

Identifies the device within the network.

---

## Subnet Mask

A subnet mask determines which part of an IP address belongs to the network and which part belongs to the host.

Example:

IP Address:
192.168.1.10

Subnet Mask:
255.255.255.0

---

## CIDR Notation

CIDR (Classless Inter-Domain Routing) is a shorter way to represent subnet masks.

Examples:

- /8
- /16
- /24

---

## Common CIDR Values

| CIDR | Subnet Mask |
|--------|------------|
| /8 | 255.0.0.0 |
| /16 | 255.255.0.0 |
| /24 | 255.255.255.0 |

---

## Real World Example

A company may divide departments into separate subnets:

- HR
- Finance
- IT
- Management

This improves organization and security.

---

## Importance in Cybersecurity

Subnetting helps:

- Segment networks
- Limit attack spread
- Improve monitoring

---

## Key Learning

Subnetting divides networks into smaller sections, improving performance, security, and management.

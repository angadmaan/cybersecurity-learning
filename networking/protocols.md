# Network Protocols

## Introduction

A protocol is a set of rules that define how devices communicate over a network.

Without protocols, devices would not be able to exchange information effectively.

---

## Why Protocols Are Important

Protocols ensure:

- Reliable communication
- Standardized data exchange
- Interoperability between systems

---

## Common Network Protocols

### HTTP

HyperText Transfer Protocol

Purpose:
Transfer web pages.

Default Port:
80

---

### HTTPS

HyperText Transfer Protocol Secure

Purpose:
Secure web communication using encryption.

Default Port:
443

---

### FTP

File Transfer Protocol

Purpose:
Transfer files between systems.

Ports:
20 and 21

---

### SMTP

Simple Mail Transfer Protocol

Purpose:
Send emails.

Port:
25

---

### POP3

Post Office Protocol Version 3

Purpose:
Receive emails.

Port:
110

---

### DNS

Domain Name System

Purpose:
Convert domain names into IP addresses.

Port:
53

---

### SSH

Secure Shell

Purpose:
Secure remote access to systems.

Port:
22

---

## Protocol Layers

Protocols operate at different layers of networking models such as:

- OSI Model
- TCP/IP Model

Each protocol performs a specific function.

---

## Secure vs Insecure Protocols

### Insecure

- HTTP
- FTP
- Telnet

### Secure

- HTTPS
- SSH
- SFTP

Secure protocols use encryption to protect data.

---

## Importance in Cybersecurity

Understanding protocols helps in:

- Packet analysis
- Threat detection
- Network troubleshooting
- Security monitoring

Tools such as Wireshark rely heavily on protocol analysis.

---

## Key Learning

Protocols define how devices communicate. Understanding major networking protocols is essential for both networking and cybersecurity professionals.

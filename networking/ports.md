# Network Ports

## Introduction

Ports are logical communication endpoints used by applications and services.

They help identify which service should receive incoming data.

---

## Why Ports Are Needed

A device can run multiple services simultaneously.

Ports allow the operating system to distinguish between these services.

Examples:

- Web Browsing
- Email
- File Transfer

---

## Common Ports

| Service | Port |
|-----------|------|
| HTTP | 80 |
| HTTPS | 443 |
| FTP Data | 20 |
| FTP Control | 21 |
| SMTP | 25 |
| DNS | 53 |
| POP3 | 110 |
| IMAP | 143 |
| SSH | 22 |

---

## Types of Ports

### Well-Known Ports

Range:

0–1023

Used by common services.

---

### Registered Ports

Range:

1024–49151

Used by specific applications.

---

### Dynamic Ports

Range:

49152–65535

Assigned temporarily by operating systems.

---

## Open and Closed Ports

### Open Port

Accepts incoming connections.

### Closed Port

Rejects incoming connections.

---

## Port Scanning

Port scanning is the process of identifying open ports on a system.

Common Tool:

- Nmap

Used for:

- Security assessment
- Network auditing
- Vulnerability discovery

---

## Importance in Cybersecurity

Ports help security professionals:

- Identify running services
- Detect vulnerabilities
- Monitor exposure
- Strengthen network security

---

## Key Learning

Ports allow multiple services to operate on a single device and are essential for network communication and cybersecurity assessments.

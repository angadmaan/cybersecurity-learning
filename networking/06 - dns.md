# Domain Name System (DNS)

## Introduction

DNS stands for Domain Name System.

It converts human-readable domain names into IP addresses that computers can understand.

Example:

google.com → 142.250.x.x

---

## Why DNS is Needed

Humans remember names easily.

Computers communicate using IP addresses.

DNS acts as a translator between the two.

---

## How DNS Works

1. User enters a website name.
2. DNS receives the request.
3. DNS finds the corresponding IP address.
4. User is connected to the website.

---

## Common DNS Components

### DNS Server

Stores DNS records and answers queries.

### DNS Resolver

Receives requests from users and finds the required information.

### DNS Record

Stores information about domains.

Examples:

- A Record
- AAAA Record
- MX Record
- CNAME Record

---

## Benefits of DNS

- Easy website access
- Faster navigation
- Simplified network management
- Supports email routing

---

## DNS Security Risks

### DNS Spoofing

Providing false DNS information to users.

### DNS Cache Poisoning

Injecting fake DNS records into cache.

### DNS Tunneling

Using DNS traffic to transfer malicious data.

---

## DNS Port

Default Port:

53

Protocols:

- UDP
- TCP

---

## Importance in Cybersecurity

DNS monitoring helps identify:

- Malicious domains
- Command-and-control servers
- Phishing activities
- Suspicious network traffic

---

## Key Learning

DNS translates domain names into IP addresses and is one of the most important services that makes internet communication possible.
